트랜잭션 = DBMS에서 데이터를 다루는 논리적인 작업의 단위.  전체가 수행되거나 전혀 수행되지 않아야 함.
시작 -> 수행 -> 부분완료 -> 버퍼내용 기록 -> 찐완료
4가지 성질 -원자성 : 모두 수행되거나 전부 수행X여야 함.
	  -일관성 : 트랜잭션 수행 전이나 후나 db는 항상 일관된 상태를 유지해야 함.
	  -고립성 : 수행 중인 트랜잭션에 다른 트랙잭션이 끼어들어 변경 중인 데이터 값을 훼손하는 일이 없어야 한다.
	  -지속성: 수행을 완료한 트랜잭션은 변경한 데이터를 영구적으로 저장해야 한다.

동시성 제어 : 트랙잭션이 동시에 수행 될 때, 일관성을 해치지 않도록 트랜잭션의 데이터 접근을 제어하는 DBMS의 기능
