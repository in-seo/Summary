★릴레이션!='관계',  테이블을 릴레이션이라고 부른다.
릴레이션 = 스키마+인스턴스    
스키마: 릴레이션이 어떻게 구성되는지, 어떤 정보를 담고 있는지에 대한 기본적인 구조
----> 릴레이션에 어떤 정보가 담길지 정의 ex)도서번호|이름|출판사|가격
--속성: 릴레이션 스키마의 열, 
--도메인: 속성이 가질 수 있는 값 타입 
--차수: 속성의 개수(열의 개수)

투플(Tuple): 릴레이션의 행( 테이블의 행) 

DB에 저장된 데이터는 결함이 없어야 하며 질의에 대해 신뢰성 있는 답을 제공 해야한다.
슈퍼키 : 투플을 유일하게 식별할 수 있는 하나의 속성, 혹은 속성의 집합.
☆후보키 : 투플을 유일하게 식별할 수 있는 속성의 최소 집합.  ex)주민번호,고객번호
						
기본키(PK) 제약조건 : 
-릴레이션 내 투플을 식별할 수 있는 고유값
- null(X), 키 값 변동 

대리키 : 기본키 사용하기 힘든 상황에 임의로 dbms에서 생성하는 키(직관적으로 파악X)
대체키 : 기본키로 선정되지 않은 후보키

외래키(FK) : 다른 릴레이션의 기본키를 참조하는 속성
-참조되는(PK) 값이 변경되면 참조하는(FK)도 따라서 변경된다. 
- null, 중복 값 허용. (당연)

도메인 무결성 제약조건 : 릴레이션 내 투플들이 각 속성의 도메인에 지정된 값만 가져야 한다.
ex) 주문 릴레이션에서 주문일자는 날짜 데이터만 사용해야 함. like 프로그래밍에서 변수형 선언
개체 무결성 제약조건(기본키 제약) : 각각의 기본키는 null값 없이 유일한 값 가짐
참조 무결성 제약조건(외래키 제약) : 자식의 외래키는 부모의 기본키와 도메인이 동일해야 한다.
ex) 외래키(3L),  기본키 (3L).  서로 변경될 때 거부되는 제약이 있음.

조인- 세타조인 : 조인에 참여하는 두 릴레이션의 속성 값을 비굫여 조건을 만족하는 투플만 반환
      - 동등조인(내부조인!) : 세타조인에서 = 연산자를 사용한 조인. (기본적인 조인임)
		↘ 릴레이션이 첫 번쨰와 두 번째 릴레이션의 차수를 합한 것과 같음..
		ex) 고객번호|이름|주소|폰|주문번호|고객번호|도서번호|판매가격|주문일자|
      - 자연조인 : 동등조인에서 같은 속성은 한번만 표시
		ex) 고객번호|이름|주소|폰|주문번호|도서번호|판매가격|주문일자|
      - 외부조인 : 자연조인 시 조인에 실패한 투플을 모두 보여주되 값이 없는 대응 속성에는
		null값을 채워서 반환한다.
-------------------------------------------------------요약----------------------------------------------------
1. 릴레이션 : 관계 데이터 모델의 핵심개념으로 행과 열로 구성된 테이블.
2. 릴레이션 스키마: 관계DB에 테이블이 어떻게 구성되는지, 어떤 정보를 담고 있는지에 대한 기본적인 구조.
3. 키       - 슈퍼키: 튜플을 유일하게 식별할 수 있는 속성의 집합.
	↘후보키: 튜플을 유일하게 식별할 수 있는 속성의 최소 집합.
	↘기본키: 여러 후보키 중 하나를 선정하여 대표로 삼는 키
	↘대리키: 직관적으로 파악할 수 없게 번호를 붙여 기본키로 삼는 것
	↘외래키: 다른 릴레이션의 기본키를 참조하는 속성
셀렉션 : σ 투플을 추출하기 위한 연산.  (투플의 조건)
프로젝션: π 릴레이션의 속성(열)을 추출하기 위한 연산
